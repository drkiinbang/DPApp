# pointcloud_viewer.html — BIM + 포인트클라우드 뷰어

BIM 모델과 포인트클라우드를 **같은 좌표계에서 겹쳐 보고**, 테스트용 **합성 스캔 데이터를 만들어 저장**할 수 있는 독립 실행형 도구입니다.

- **설치 불필요**: 파일 하나만 브라우저로 열면 됩니다. 외부 라이브러리·인터넷 연결·서버 모두 불필요합니다(WebGL과 GLB 파서를 자체 구현).
- **데이터가 밖으로 나가지 않음**: 모든 처리는 브라우저 안에서만 일어납니다.

## 왜 쓰나요

- **정합 전에 눈으로 확인**: BIM과 스캔이 애초에 얼마나 어긋나 있는지, ICP를 돌리기 전에 눈으로 봅니다.
- **정합 결과 확인**: ICP가 만든 `*_icp_aligned.las`를 BIM과 함께 열어 실제로 잘 겹쳐졌는지 봅니다.
- **테스트 데이터 생성**: 실측 대용량 데이터 없이, BIM에서 **정답(ground truth)을 아는** 합성 스캔을 만들어 ICP가 그 오차를 제대로 복원하는지 검증합니다.

> **중요**: 합성 스캔 생성 방식은 Master의 `gensynth` 명령과 **완전히 동일한 알고리즘**입니다(면적 비례 barycentric 격자 샘플링 + 중심 기준 회전/이동/노이즈 주입). 즉 이 뷰어에서 만든 데이터로 검증한 결과는 `gensynth`로 만든 것과 같은 의미를 갖습니다. 관련 코드: `include/bimtree/PseudoPointGenerator.hpp`, `MasterApp/MasterApplication_ICP.cpp`의 `runGenerateSyntheticPointCloud()`.

## 여는 방법

`tools/pointcloud_viewer.html` 파일을 **더블클릭**하거나 브라우저(Chrome/Edge 등)로 드래그하면 바로 실행됩니다. 별도 설치·서버가 필요 없습니다.

## 지원 파일 형식

| 종류 | 지원 | 비고 |
|---|---|---|
| BIM | `.glb` | **단일 파일(.glb)만 지원.** 외부 `.bin`을 참조하는 `.gltf`는 지원하지 않습니다. 여러 개를 한 번에 선택하면 하나의 BIM으로 합쳐서 표시합니다(부재별 파일이 나뉘어 있는 `data/10glb/` 같은 구조에 적합). |
| 포인트클라우드 | `.las`, `.xyz`, `.pts`, `.txt`, `.csv`, `.asc` | 대용량 파일은 "최대 포인트" 예산을 넘으면 자동으로 솎아내어(서브샘플링) 표시합니다. |
| 포인트클라우드 | `.laz` ❌ | **지원하지 않습니다.** `.las`로 변환한 뒤 열어주세요. (C++ 엔진은 `.laz`를 정상 지원하지만, 이 뷰어는 외부 라이브러리 없이 만들어 LASzip 압축 해제를 넣지 않았습니다.) |

파일 선택 버튼 대신 **창에 드래그&드롭** 해도 됩니다. `.glb`는 BIM으로, 나머지는 포인트클라우드로 자동 분류됩니다.

## 기본 사용법

### 1. 데이터 불러오기

- **`BIM 열기 (.glb, 여러 개 선택 가능)`** — 예: `data/10glb/` 안의 `.glb` 파일들을 전부 선택
- **`포인트클라우드 열기`** — 예: 실측 `.las` 또는 ICP 결과 `*_icp_aligned.las`

BIM과 포인트클라우드를 **둘 다 열면 같은 좌표계에 겹쳐서** 표시됩니다(먼저 연 쪽의 원점을 공유하므로 좌표가 어긋나지 않습니다). 시야는 자동으로 전체가 보이게 맞춰집니다.

- **`모두 지우기`** — BIM/포인트클라우드를 모두 비웁니다.

### 2. 화면 조작

| 조작 | 동작 |
|---|---|
| 드래그 | 회전 |
| 휠 | 확대/축소 |
| 우클릭 드래그 (또는 Shift+드래그) | 이동(pan) |
| 더블클릭 | 전체 보기 |

`시점` 섹션의 `전체 보기` / `위에서` / `정면` 버튼으로 정해진 시점으로 즉시 이동할 수 있고, `PNG 저장`으로 현재 화면을 이미지로 저장합니다.

### 3. 표시 옵션

- **BIM**: `BIM 표시` 체크 해제로 숨기기, `불투명도` 슬라이더로 반투명 조절 (BIM 안쪽의 포인트를 보고 싶을 때 유용).
- **색상**: `RGB(파일 색상)` / `높이(Z)` / `강도(Intensity)` / `단색` 중 선택. 파일에 색상이 없으면 자동으로 적절한 모드가 선택됩니다.
- **포인트 크기**, **원근 크기 감쇠**, **원형 포인트**, **바닥 그리드**, **배경**(어두움/검정/밝음)도 조절 가능합니다.

## 합성 스캔 생성 (테스트용 핵심 기능)

BIM 표면에서 가상의 스캔 포인트를 만들고, **알고 있는 오차**를 일부러 주입합니다. ICP가 그 오차를 정확히 복원하는지 검증하는 데 씁니다.

**절차**: `BIM 열기`로 BIM을 먼저 불러오면 `합성 스캔 생성` 버튼이 활성화됩니다. 파라미터를 입력하고 버튼을 누르세요.

| 파라미터 | 의미 | 권장 |
|---|---|---|
| `격자 간격(m)` | 표면에 점을 얼마나 촘촘히 뿌릴지 (작을수록 점 많음) | `0.02` (기본). 점이 너무 많으면 키우세요 |
| `노이즈 σ(m)` | 측정 오차 시뮬레이션 (가우시안) | `0`이면 완벽한 데이터, 실제와 비슷하게 하려면 `0.005`~`0.01` |
| `이동 X/Y/Z(m)` | 주입할 평행이동 오차 | 예: `0.02`, `0.03`, `0.01` |
| `회전(Z축, °)` | 주입할 회전 오차 (BIM 전체 bbox 중심 기준) | 예: `3` |

생성 후 **`주입한 정답(ground truth)`** 정보가 화면에 표시됩니다 — 회전각·이동량·회전 중심·점 개수. **ICP 결과가 이 값을 복원해야 정상**입니다.

> **팁**: 회전을 `3`도 주입했다면, ICP 결과 `_alignment.json`의 부재별 `twist_angle_deg`가 대략 `3`에 가깝게 나와야 합니다.

## 저장하기

`포인트클라우드 정보` 섹션의 `저장 형식`을 고르고 **`포인트클라우드 저장`** 버튼을 누릅니다. 합성 스캔이든 불러온 실측 데이터든 현재 표시 중인 포인트클라우드를 저장합니다.

| 형식 | 용도 |
|---|---|
| **`.las` (바이너리)** | **실제 분산 ICP 파이프라인에 그대로 투입 가능.** 아래 "실전 활용" 참고 |
| `.xyz` (텍스트) | CloudCompare 등 다른 도구에서 열어보기 |

> `.las` 저장은 1mm 정밀도로 양자화됩니다(좌표 스케일 0.001). 밀리미터 이하 정밀도가 필요한 용도가 아니면 문제없습니다.

## 실전 활용: 합성 데이터로 ICP 검증하기

1. 이 뷰어로 `data/10glb`를 열고, 회전 `3`도 + 이동 `0.02/0.03/0.01`을 주입해 합성 스캔 생성 → 표시된 **ground truth 기록**
2. `.las`로 저장 (예: `synth_test.las`)
3. Master/Slave를 띄우고 REST로 ICP 실행:
   ```
   curl -X POST http://localhost:8081/api/icp/start ^
     -H "Content-Type: application/json" ^
     -d "{\"las_file\": \"...\\synth_test.las\", \"bim_folder\": \"...\\data\\10glb\"}"
   ```
4. 완료 후 `*_alignment.json`의 `twist_angle_deg` / `combined_transform`이 1단계에서 기록한 정답을 복원했는지 확인
5. 결과 `*_icp_aligned.las`를 이 뷰어에서 BIM과 함께 열어 **눈으로도** 잘 겹쳤는지 확인

> 같은 검증을 콘솔에서 하려면 Master의 `gensynth` 명령을 쓰면 됩니다(동일 알고리즘). 자세한 절차는 [doc/[DPApp] handover/06_상세_테스트_절차서_비전문가용.md](../doc/%5BDPApp%5D%20handover/06_%EC%83%81%EC%84%B8_%ED%85%8C%EC%8A%A4%ED%8A%B8_%EC%A0%88%EC%B0%A8%EC%84%9C_%EB%B9%84%EC%A0%84%EB%AC%B8%EA%B0%80%EC%9A%A9.md) §11 참고.

## 문제 해결

| 증상 | 원인/해결 |
|---|---|
| `.laz` 파일이 안 열림 | 의도된 제약입니다. `.las`로 변환 후 열어주세요. |
| `.gltf`가 안 열림 | 외부 `.bin`을 참조하는 `.gltf`는 미지원. `.glb`(단일 파일)를 쓰세요. |
| 대용량 파일이 다 안 보임 | `최대 포인트` 예산을 넘으면 자동 서브샘플링됩니다. 옵션에서 예산을 키우거나(메모리 주의), 그대로 두어도 형상 확인에는 지장 없습니다. |
| `합성 스캔 생성` 버튼이 비활성 | BIM을 먼저 불러와야 활성화됩니다. |
| 포인트가 안 보임 | `전체 보기`(더블클릭)로 시야를 맞춰보세요. BIM에 가려졌다면 `BIM 표시`를 끄거나 `불투명도`를 낮추세요. |
| 화면이 검게만 나옴 | 브라우저가 WebGL을 지원하는지 확인하세요(최신 Chrome/Edge 권장). |
