# 03. 빌드 가이드 (소스 코드를 실행 파일로 만들기)

"빌드(Build)"란, 사람이 작성한 소스 코드(텍스트)를 컴퓨터가 실행할 수 있는 프로그램(.exe)으로 변환하는 과정입니다. 요리 재료(소스코드)를 요리(실행파일)로 만드는 과정이라고 생각하면 됩니다.

## 1. 준비물 (사전 설치 필요)

| 항목 | 용도 | 확인 방법 |
|---|---|---|
| **Visual Studio 2017 이상** (Community 버전도 가능, 무료) | Windows에서 C++ 빌드 | 설치 시 "C++를 사용한 데스크톱 개발" 워크로드 체크 |
| (선택) **CMake** | Linux/Mac에서 빌드하거나 자동화된 빌드를 하고 싶을 때 | `cmake --version` 으로 확인 |

> 이 프로젝트는 기본적으로 **Windows + Visual Studio** 환경을 기준으로 만들어졌습니다. (`bin/` 폴더의 결과물이 `.exe`, `.dll`임)

## 2. Visual Studio로 빌드하기 (가장 쉬운 방법, 권장)

1. 탐색기에서 `F:\repository\DPApp_II\DPApp.sln` 파일을 더블클릭해서 엽니다.
2. Visual Studio가 열리면, 화면 위쪽 도구막대에서 **빌드 구성**을 선택합니다.
   - 처음 테스트할 때는 `Debug` (느리지만 문제 발생 시 원인 파악이 쉬움)
   - 실제 사용할 때는 `Release` (빠름)
3. 화면 오른쪽의 **솔루션 탐색기**에 아래와 같은 여러 프로젝트가 보입니다.
   - `MasterApp`, `SlaveApp` (메인 프로그램)
   - `BimImport`, `LasImport` (파일 읽기용 라이브러리)
   - `MasterLauncher`, `SlaveAgent` (원격 실행 보조 프로그램)
   - `TestTaskManager`, `TestLasImport`, `TestLaslibWrapper`, `TestGltfImport` (테스트)
4. 메뉴에서 **빌드(Build) → 솔루션 다시 빌드(Rebuild Solution)** 를 클릭합니다. (단축키: `Ctrl+Shift+B` 는 일반 빌드, 다시빌드는 메뉴에서 직접 선택)
5. 하단의 "출력(Output)" 창에 빌드 로그가 나옵니다. 마지막에 `0개 실패` 처럼 나오면 성공입니다.
6. 빌드가 끝나면 `F:\repository\DPApp_II\bin\` 폴더에 `MasterApp.exe`, `SlaveAppd.exe` 등의 실행 파일이 새로 만들어지거나 갱신됩니다.

### 빌드가 실패했을 때 흔한 원인

| 오류 메시지(키워드) | 원인 | 해결 방법 |
|---|---|---|
| `cannot open include file` (헤더를 못 찾음) | vendors(Eigen3, LAStools 등) 경로가 깨졌거나 누락 | `vendors/` 폴더가 그대로 있는지 확인. 프로젝트 속성의 "추가 포함 디렉터리" 확인 |
| `LNK2019` / `unresolved external symbol` (링크 오류) | 라이브러리(.lib) 파일을 못 찾음 | 프로젝트 속성의 "추가 라이브러리 디렉터리"/"추가 종속성" 확인 |
| `이 플랫폼 도구 집합이 설치되어 있지 않습니다` | Visual Studio 버전이 다름 | 솔루션 우클릭 → "플랫폼 도구 집합 재대상"으로 설치된 버전으로 변경 |

## 3. CMake로 빌드하기 (선택, Linux/Mac 등)

```bash
cd F:\repository\DPApp_II
cmake -S . -B build
cmake --build build --config Release
```

또는 Makefile을 직접 사용:

```bash
cd F:\repository\DPApp_II
make all
```

> Windows에서는 1번(Visual Studio) 방법을 권장합니다. CMake/Makefile은 보조 수단입니다.

## 4. 빌드 결과물 확인

빌드가 끝나면 아래 파일들이 `bin/` 폴더에 있어야 합니다.

```
bin/
├── MasterApp.exe      ← Master 프로그램
├── SlaveAppd.exe      ← Slave 프로그램 (Debug 빌드는 보통 이름 끝에 'd'가 붙습니다)
├── MasterLauncher.exe
├── SlaveAgent.exe
├── BimImport.dll
├── LasImport.dll
└── Test*.exe          ← 각종 테스트 실행 파일
```

이 파일들이 모두 만들어졌다면 빌드는 성공입니다. 다음으로 [04_테스트_가이드.md](04_테스트_가이드.md) 에서 정상 동작을 검증하는 방법을 안내합니다.
