# 01. 프로젝트 개요 및 구조

## 1. 이 프로그램은 무엇인가?

### 1.1 풀어서 설명

건설/시공 현장에는 두 가지 종류의 "3차원 데이터"가 존재합니다.

| 데이터 종류 | 무엇인가 | 이 프로그램에서의 파일 형식 |
|---|---|---|
| **포인트클라우드(Point Cloud)** | 레이저 스캐너(LiDAR)로 실제 현장을 스캔해서 얻은 수백만~수십억 개의 점(점 하나하나가 X, Y, Z 좌표를 가짐) | `.las`, `.xyz`, `.pts` |
| **BIM 메시(Mesh)** | 설계 단계에서 컴퓨터로 만든 3D 건물 모델 (벽, 기둥, 슬래브 등을 삼각형 면들로 표현) | `.gltf`, `.glb` |

문제는, 스캐너로 찍은 점들과 설계 모델은 **처음에는 서로 다른 좌표계(위치, 회전, 크기 기준)** 를 가지고 있어서 그냥 겹쳐 보면 위치가 어긋나 있습니다.

**DPApp_II는 이 두 데이터를 자동으로 분석해서, 서로 가장 잘 맞는 위치로 점들을 이동/회전시켜 정확히 겹치도록 만들어주는 프로그램**입니다. 이 작업을 전문 용어로 **ICP 정합(Iterative Closest Point Registration)**이라고 부릅니다.

추가로, 포인트클라우드의 양이 매우 많기(기가바이트 단위) 때문에 **한 대의 컴퓨터(Master)가 작업을 여러 조각으로 나누고, 여러 대의 컴퓨터(Slave)에게 나눠서 동시에 처리시킨 뒤 결과를 다시 모으는** 분산 처리 구조로 만들어져 있습니다.

### 1.2 비유로 이해하기

- **Master(마스터)** = 식당의 "주방장". 손님 주문(작업)을 받아서 일을 잘게 나눠 여러 요리사(Slave)에게 분배하고, 완성된 요리(결과)를 모아서 손님에게 내보냅니다.
- **Slave(슬레이브)** = "요리사". 주방장이 시킨 부분 작업만 처리해서 다시 돌려줍니다.
- **ICP 정합** = 흩어진 퍼즐 조각(스캔 데이터)을 원본 그림(설계 모델) 위에 딱 맞게 움직여서 맞추는 작업.

## 2. 전체 시스템 그림

```
┌─────────────────────────────────────────────────┐
│  사용자 (사람)                                    │
│  - Master 콘솔 창에 명령어를 입력                  │
│  - 또는 REST API(인터넷 주소 형태의 명령)로 요청     │
└────────────────────┬──────────────────────────────┘
                      │
            ┌─────────▼──────────┐
            │   MasterApp.exe    │   ← "주방장" 프로그램
            │  - 작업을 잘게 나눔   │
            │  - Slave에게 분배    │
            │  - 결과를 모아 합침   │
            └─────────┬──────────┘
                      │ (네트워크 통신, TCP/IP)
        ┌─────────────┼─────────────┐
        ▼             ▼             ▼
 ┌─────────────┐┌─────────────┐┌─────────────┐
 │SlaveApp.exe ││SlaveApp.exe ││SlaveApp.exe │  ← "요리사" 프로그램
 │ (컴퓨터 #1) ││ (컴퓨터 #2) ││ (컴퓨터 #3) │     (여러 대 PC에서 각각 실행)
 └─────────────┘└─────────────┘└─────────────┘
        │             │             │
        └─────────────┼─────────────┘
                      ▼
        입력: 포인트클라우드(.las/.xyz) + BIM 메시(.glb)
        출력: 정렬이 끝난 포인트클라우드(.las) + 변환 정보(위치를 얼마나 옮겼는지)
```

> 참고: 컴퓨터 한 대만 있어도 동작합니다. 그 경우 Master와 Slave를 같은 PC에서 둘 다 실행하면 됩니다. "여러 대"는 데이터가 매우 크고 빠른 처리가 필요할 때를 위한 선택 사항입니다.

## 3. 폴더 구조와 각 폴더의 역할

## 4. 핵심 개념 미니 사전

처음 보면 헷갈리는 용어들을 미리 정리해 둡니다.

| 용어 | 쉬운 설명 |
|---|---|
| **포인트클라우드 (Point Cloud)** | 점들의 집합. 각 점은 (X, Y, Z) 좌표를 가짐. 레이저 스캐너의 결과물. |
| **메시 (Mesh)** | 점들을 삼각형으로 이어서 만든 표면. BIM(설계) 모델은 메시로 표현됨. |
| **ICP (Iterative Closest Point)** | "반복적으로 가장 가까운 점을 찾아서 맞춰나가는" 정합 알고리즘. 한 번에 딱 맞추는 게 아니라, 조금씩 옮기고 — 다시 가까운 점을 찾고 — 또 조금씩 옮기고를 반복(iterate)해서 점점 정확하게 맞춰갑니다. |
| **변환 행렬 (Transform Matrix)** | "얼마나 옮기고(이동), 얼마나 돌리고(회전), 얼마나 늘리거나 줄였는지(크기)"를 숫자 4x4 표로 표현한 것. ICP의 최종 결과물입니다. |
| **RMSE** | 정합이 얼마나 잘 됐는지를 나타내는 오차 점수. 작을수록 잘 맞은 것. (단위는 보통 미터) |
| **다운샘플링 (Downsampling)** | 점이 너무 많으면 계산이 느려지므로, 일부 점만 골라서 계산을 빠르게 하는 것. (예: 10개 중 1개만 사용) |
| **이상치 (Outlier)** | 잘못 측정되었거나 비교 대상이 없는 점. 정합 품질을 떨어뜨리므로 자동으로 걸러냄(제거). |
| **Master / Slave** | Master는 작업을 나눠주는 쪽(주방장), Slave는 실제 계산을 수행하는 쪽(요리사). |
| **청크 (Chunk)** | 큰 작업을 작게 나눈 조각 하나. Master가 포인트클라우드를 청크 단위로 잘라서 여러 Slave에게 나눠줍니다. |
| **REST API** | 웹 브라우저 주소창에 입력하는 것과 비슷한 방식(http://...)으로 프로그램에 명령을 내리는 방법. 사람이 직접 콘솔에 타이핑하지 않고, 다른 프로그램이 자동으로 명령을 보낼 때 사용. |

## 5. 기술적으로 사용된 것들 (참고용)

- 프로그래밍 언어: **C++17**
- 운영체제/개발환경: **Windows + Visual Studio**
- 수학 연산: **Eigen3** (행렬, SVD 연산)
- 최근접점 탐색: **nanoflann** (KD-Tree, 빠르게 "가장 가까운 점"을 찾는 자료구조)
- LAS 파일 처리: **LAStools**
- 네트워크: **TCP/IP 소켓 통신** (직접 구현)
- 테스트 프레임워크: **Google Test**
