# 02. 핵심 기능 명세

이 문서는 "실제로 무엇이 개발되어 있는가"를 기능 단위로 정리한 것입니다. 코드를 직접 읽지 않아도 이 표만 보고 "이 프로그램이 뭘 할 수 있는지"를 파악할 수 있도록 작성했습니다.

## 1. 지원하는 작업 종류 (Task Type)

DPApp_II는 "작업(Task)"이라는 단위로 일을 처리합니다. 현재 구현된 작업 종류는 다음과 같습니다.
(코드 위치: `include/PointCloudTypes.h`, `include/IcpTypes.h`)

| 작업 이름 | 코드 번호 | 한 줄 설명 |
|---|---|---|
| CONVERT_PTS | 1 | 포인트클라우드 형식 변환 (예: .xyz → .las) |
| BIM_DISTANCE_CALCULATION | 2 | 포인트와 BIM 메시 사이의 거리 계산 (시공 오차 검사 등에 활용) |
| BIM_PC2_DIST | 3 | PC2 포맷 포인트클라우드와 BIM 간 거리 계산 |
| TEST_ECHO | 10 | (테스트용) 보낸 데이터를 그대로 돌려받기 |
| TEST_COMPUTE | 11 | (테스트용) 간단한 계산 결과 검증 |
| TEST_DELAY | 12 | (테스트용) 의도적으로 지연시켜 타임아웃/대기 동작 확인 |
| TEST_FAIL | 13 | (테스트용) 의도적으로 실패를 발생시켜 오류 처리 확인 |
| **ICP_COARSE_ALIGNMENT** | 20 | ICP **전역(대략) 정합** — 큰 틀에서 위치를 맞추는 1차 작업 (Master에서 직접 수행) |
| **ICP_FINE_ALIGNMENT** | 21 | ICP **정밀 정합** — 1차로 맞춘 뒤, 여러 Slave가 나눠서 정밀하게 다듬는 작업 |
| **ICP_APPLY_TRANSFORM** | 22 | 계산이 끝난 "변환 행렬"을 실제 포인트클라우드 전체에 적용해서 파일로 저장 |

> 핵심 기능은 마지막 3개(ICP 관련)입니다. 위쪽의 TEST_* 항목들은 시스템이 정상 동작하는지 점검하기 위한 자가진단용 작업입니다.

## 2. ICP 정합 알고리즘 상세 (가장 중요한 부분)

코드 위치: `include/bimtree/IcpCore.hpp`

ICP는 다음 단계를 반복(iterate)합니다. 각 단계에 대응하는 기능이 모두 구현되어 있습니다.

```
[1단계] 가상 포인트 생성
   BIM 메시(삼각형들)는 "점"이 아니라 "면"이므로, 비교를 위해
   메시 표면 위에 가상의 점들을 일정 간격(그리드)으로 흩뿌립니다.
   → PseudoPointGenerator.hpp, PseudoPtsInMesh.hpp

[2단계] 다운샘플링 (속도 향상)
   점이 너무 많으면 느리므로 일부만 추려냅니다.
   - downsampleUniform(): 일정 비율로 추리기 (예: 10개 중 1개)
   - downsampleVoxelGrid(): 공간을 격자(voxel)로 나눠 칸마다 1개씩 추리기

[3단계] 최근접점(대응점) 찾기
   스캔한 점 하나마다 "BIM 가상점 중 가장 가까운 점"을 찾습니다.
   KD-Tree(nanoflann)라는 빠른 탐색 자료구조를 사용합니다.
   - findCorrespondences(): 기본 최근접점 탐색
   - findCorrespondencesWithNormals(): 표면이 향하는 방향(법선)까지 고려해서
     더 정확한 대응점만 선택 (엉뚱한 면에 매칭되는 오류 방지)

[4단계] 이상치(틀린 매칭) 제거
   - rejectOutliersMAD(): 통계적으로 "너무 먼" 대응점들을 자동으로 걸러냄
   - rejectOutliersPercentile(): 거리 상위 몇 %를 잘라내는 방식

[5단계] 최적의 이동/회전 계산
   - estimateRigidTransformSVD(): "이렇게 옮기고 돌리면 가장 잘 맞는다"는
     이동+회전 값을 수학적으로(SVD) 계산
   - transformPointCloud(): 계산된 값으로 점들을 실제로 이동/회전시켜봄

[6단계] 오차 측정
   - computeRMSE(): 지금까지 맞춘 정도(오차)를 숫자로 측정

[1~6단계를 설정된 횟수만큼 반복 → 오차가 충분히 작아지면(수렴) 중단]
   → runIcp() 함수가 이 전체 반복 루프를 담당

[7단계] (분산 처리인 경우) 여러 Slave의 결과를 합치기
   - aggregateResultsWeighted(): 각 Slave가 계산한 결과를 가중 평균으로 통합
```

### 2.1 ICP 설정값 (사용자가 조절 가능한 옵션)

코드 위치: `include/IcpTypes.h` (IcpConfig 구조체)

| 설정 이름 | 기본값 | 의미 | 값을 바꾸면? |
|---|---|---|---|
| `maxIterations` | 50 | 최대 반복 횟수 | 늘리면 더 정확해질 수 있지만 느려짐 |
| `convergenceThreshold` | 1e-6 | "이 정도면 충분히 맞았다"고 보는 오차 기준 | 작게 하면 더 정밀해지지만 시간이 더 걸림 |
| `maxCorrespondenceDistance` | 1.0 (미터) | 이 거리보다 먼 점은 짝(대응점)으로 인정하지 않음 | 처음 위치가 많이 어긋나 있다면 이 값을 키워야 함 |
| `outlierRejectionThreshold` | 2.5 | 이상치로 판단하는 통계적 기준(배수) | 작게 하면 더 엄격하게 걸러냄 |
| `downsampleRatio` | 10 | 10개 중 1개만 사용 | 크게 하면 빨라지지만 정밀도는 떨어짐 |
| `useNormalFiltering` | true | 법선 방향까지 고려할지 | 꺼두면 더 빠르지만 오매칭 가능성 증가 |
| `normalAngleThreshold` | 45.0 (도) | 법선 방향이 이 각도 이상 다르면 대응점 제외 | - |
| `minCorrespondences` | 100 | 최소 대응점 수 (이보다 적으면 정합 실패로 판단) | - |
| `pseudoPointGridSize` | 0.1 (미터) | BIM 메시에서 가상점을 만드는 격자 간격 | 작게 하면 더 정밀하지만 가상점 수가 늘어나 느려짐 |

> 이 값들은 보통 `config/master.ini` 또는 REST API 요청(JSON)으로 전달합니다. 자세한 방법은 [05_사용자_매뉴얼.md](05_사용자_매뉴얼.md)에서 다룹니다.

## 3. 입출력 파일 포맷

| 구분 | 포맷 | 처리 코드 |

## 4. Master ↔ Slave 통신 기능

코드 위치: `include/NetworkManager.h`

| 메시지 종류 | 용도 |
|---|---|
| HANDSHAKE | Slave가 Master에 처음 연결될 때 "나 연결됐어요" 인사 |
| SLAVE_REGISTER | Slave가 자신의 정보(이름/성능 등)를 Master에 등록 |
| TASK_ASSIGNMENT | Master가 작업 조각을 Slave에게 보냄 |
| TASK_RESULT | Slave가 계산을 끝내고 결과를 Master에게 돌려줌 |
| HEARTBEAT | "나 아직 살아있어요"를 주기적으로 알리는 신호 (끊긴 Slave를 감지하기 위함) |
| PAUSE / RESUME | 처리를 일시중지/재개 |
| SHUTDOWN | Slave를 안전하게 종료시키는 명령 |

이 기능 덕분에 **Slave 컴퓨터가 중간에 꺼지거나 네트워크가 끊겨도 Master가 이를 감지**할 수 있습니다.

## 5. REST API (외부 프로그램 연동용 기능)

코드 위치: `MasterApplication_ICP.cpp`

Master를 실행해두면, 웹 요청(HTTP)으로도 명령을 내릴 수 있습니다. 예를 들어 다른 회사 시스템(웹사이트, 다른 프로그램)에서 자동으로 ICP 작업을 요청하고 결과를 받아갈 수 있습니다.

| 동작 | 방식 | 주소(경로) | 설명 |
|---|---|---|---|
| ICP 작업 시작 | POST | `/api/icp/start` | 파일 경로와 설정값을 보내면 작업이 시작됨 |
| 작업 목록 조회 | GET | `/api/icp/jobs` | 지금까지 요청된 모든 작업 목록 |
| 작업 상태 조회 | GET | `/api/icp/jobs/{id}` | 특정 작업이 진행 중/완료/실패인지 확인 |
| 결과(변환행렬) 조회 | GET | `/api/icp/jobs/{id}/result` | 정합 결과 숫자값 받기 |
| 작업 취소 | POST | `/api/icp/jobs/{id}/cancel` | 진행 중인 작업 중단 |
| 통계 조회 | GET | `/api/icp/jobs/{id}/stats` | 반복 횟수, 오차 추이 등 상세 통계 |

실제 사용 예시는 [05_사용자_매뉴얼.md](05_사용자_매뉴얼.md) 의 "REST API로 사용하기" 항목을 참고하세요.
